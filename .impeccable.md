## Design Context

### Users
A small admin team managing SV Towers — a commercial property in India. They use the app on both desktop and mobile to create and send monthly rent invoices, electricity bills, and water bills to tenants. The workflow is repetitive and monthly, so speed and clarity matter most. Users are not highly technical — the interface should feel obvious and require minimal training.

### Brand Personality
**Professional, Reliable, Warm**

The app handles real money and legal documents (GST invoices), so it must feel trustworthy and business-appropriate. At the same time, it should feel approachable — not cold or corporate. Think: a well-organized accounting tool that doesn't intimidate.

### Aesthetic Direction
- **Visual tone:** Warm and professional. Shift from the current cool blue palette toward warmer tones — amber, warm gray, soft earth tones — while keeping the interface clean and structured.
- **Theme:** Light mode with warm neutrals. White/cream card surfaces, warm gray backgrounds, amber or warm-toned accents for primary actions.
- **Typography:** System UI stack. Clear hierarchy with bold headings, readable body text. No decorative fonts.
- **Icons:** Lucide React — keep existing icon usage consistent.
- **Cards & surfaces:** White cards with subtle borders and soft shadows. Generous padding, rounded corners (lg/2xl).
- **Anti-references:** Avoid playful/startup aesthetics (no gradients, illustrations, emoji, or casual tone). Avoid dense/complex data layouts — keep screens breathable with clear visual hierarchy.

### Design Principles

1. **Clarity over cleverness** — Every screen should communicate its purpose in under 2 seconds. Labels, headings, and actions should be immediately understandable. No ambiguity.

2. **Warm professionalism** — The interface should feel like a trusted business tool, not a consumer app. Use warm tones and generous spacing to keep it approachable without being casual.

3. **Mobile-first efficiency** — The admin team frequently uses this on mobile. Touch targets should be generous, forms should be streamlined, and key actions (create invoice, send bill) should be reachable in minimal taps.

4. **Consistent structure** — Lists, forms, and detail views should follow repeatable patterns. If a user learns how invoices work, electricity and water bills should feel identical in structure.

5. **Status at a glance** — Invoice/bill status (Draft, Sent, Failed) should be visually distinct and scannable. Use color coding and clear badges so the team can quickly spot what needs attention.

### Accessibility
- Target WCAG AA compliance
- Minimum 4.5:1 contrast ratio for normal text, 3:1 for large text
- All interactive elements must be keyboard accessible
- Proper ARIA labels on navigation, forms, and status indicators
- Touch targets minimum 44x44px on mobile
